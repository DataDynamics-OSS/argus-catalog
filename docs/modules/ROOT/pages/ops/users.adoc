= 사용자 · 권한

사용자 계정과 역할 기반 권한 관리. 인증 자체(로그인 흐름·SSO 설정·토큰
정책)는 xref:ops/authentication.adoc[인증] 에서 다룬다.

== 진입 경로

좌측 사이드바 *Administration → Users* (`/dashboard/users`). `argus-admin`
역할만 접근 가능.

== 사용자 관리

[cols="1,3"]
|===
| 동작 | 위치

| 등록
| Users 페이지의 *+ Add User*. 사용자명 · 이메일 · 비밀번호 · 역할 지정.
| 수정
| 행 클릭 → 상세 → 편집.
| 활성화 / 비활성화
| 상세 화면의 토글.
| 비밀번호 재설정
| 상세 화면의 *Reset Password*. Keycloak 모드면 Keycloak 에 위임.
| 삭제
| 상세 → *Delete*. 소프트 삭제, 감사 로그 보존.
|===

== 역할

기본 제공되는 시스템 역할 3 종.

[cols="1,3"]
|===
| 역할 | 권한 범위

| `argus-admin`
| 전체 권한 — 사용자 · 설정 · 모든 객체 CRUD.
| `argus-superuser`
| 관리 권한 — 사용자 관리 제외 거의 모든 작업.
| `argus-user`
| 읽기 기본 + 본인이 소유한 데이터셋 / 모델의 수정.
|===

`argus-*` 역할이 *하나도 없는* 사용자는 로그인이 거부된다 (403).

Keycloak 모드에서는 위 역할이 Realm Role 로 매핑된다. 신규 사용자는 default
role 로 `argus-user` 가 자동 부여되며, `argus-admin` / `argus-superuser` 는 수동
부여한다. 상세 절차는 xref:ops/authentication.adoc#_keycloak_수동_설정[인증 →
Keycloak 수동 설정] 참조.

== 그룹

Keycloak 모드에서는 Keycloak 의 그룹 기능을 사용한다. 그룹에 역할을 부여하면
멤버에게 일괄 상속된다. UI 에서 그룹 관리는 Keycloak Admin 콘솔에 위임.
