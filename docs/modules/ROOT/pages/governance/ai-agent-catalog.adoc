= AI Agent 카탈로그

AI 에이전트를 등록·거버넌스한다. 도구/MCP, 리니지, 버전, 평가, 미터링을 한 곳에서
관리하여 사내 AI 에이전트의 가시성과 통제를 확보한다.

== 개요

AI Agent 카탈로그는 사내 AI 에이전트(LLM 기반 자동화·어시스턴트 등)를 등록하고 다음을 제공한다.

* **등록·버전 관리** — 에이전트 등록, 버전 이력, 상태 이력
* **도구·MCP** — 에이전트가 사용하는 도구(tool)와 MCP 서버 등록
* **리니지** — 에이전트가 의존하는 데이터셋·모델·API 와의 관계
* **평가(eval)** — 에이전트 품질 평가 기록
* **미터링** — 호출(invocation) 기반 지연·토큰·성공률 지표
* **에이전트 카드·정책** — 에이전트 카드(요약)와 정책 번들(policy bundle)
* **Hook 이벤트** — 외부 에이전트가 보고하는 이벤트 수집

[NOTE]
====
외부 독립 에이전트(`agent/ serve`)는 자기 자신을 이 카탈로그에 등록하고 호출 지표를
보고(self-telemetry)할 수 있다. xref:ai/assistant.adoc[AI 어시스턴트] 와 연계된다.
====

== 주요 엔드포인트

기본 경로는 `/api/v1/ai-agents` 이다.

[cols="2,3",options="header"]
|===
| 엔드포인트 | 설명

| `POST /ai-agents` | 에이전트 등록
| `GET /ai-agents` | 목록·검색
| `GET /ai-agents/{name}` | 상세 조회
| `POST /ai-agents/{name}/tools` | 도구 등록
| `POST /ai-agents/{name}/mcp-servers` | MCP 서버 등록
| `POST /ai-agents/{name}/lineage` | 리니지 등록
| `GET·POST /ai-agents/{name}/versions` | 버전 조회/생성
| `GET /ai-agents/{name}/status-history` | 상태 이력
| `GET·POST /ai-agents/{name}/evals` | 평가 조회/등록
| `POST /ai-agents/{name}/invocations` | 호출 기록 ingest
| `GET /ai-agents/{name}/metering` | 미터링 지표(지연·토큰·성공률)
| `GET /ai-agents/{name}/card` | 에이전트 카드
| `GET /ai-agents/{name}/policy` | 정책 번들
| `GET·POST /ai-agents/{name}/hook-events` | Hook 이벤트
|===
