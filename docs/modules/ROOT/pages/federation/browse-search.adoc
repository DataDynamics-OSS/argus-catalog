= 탐색 · 통합 검색 · 리니지

== 탐색 (Browse)

페더레이션 > *탐색* 탭은 미러(HARVEST 복제본)를 둘러본다. *좌측 트리 + 우측 그리드* 의
마스터-디테일 구조다.

* *좌측 트리* — 인스턴스 → 데이터소스. 데이터소스를 클릭하면 우측에 그 데이터셋이 표시된다.
* *우측 그리드* — 선택한 데이터소스의 데이터셋 목록. 컬럼은 sidebar > 데이터셋과 동일하게
  이름·환경·필드 수·등록일·동기화일. *이름을 클릭* 하면 전체화면 상세로 이동한다.
* *이름 표시* — 논리명(`display_name`, 예: 배우)이 있으면 그대로, 없으면 물리명에서 경로를 뗀
  테이블명(예: `sakila.film` → `film`).
* *페이지네이션* — 우측 그리드는 페이지당 행 수 선택 + 번호 페이저(sidebar 데이터셋과 동일 UI).
* 상단 검색창으로 데이터셋 이름/설명을 필터해 트리를 좁힐 수 있다.

== 통합 검색 (Search)

페더레이션 > *통합 검색* 은 로컬 + 미러 + LIVE peer 를 한 번에 검색한다.

* `HARVEST`·`HYBRID` peer 는 로컬 미러(pgvector + 키워드)로, `LIVE` peer 는 요청 시 fan-out 으로 라우팅.
* 도달 실패한 LIVE peer 는 `instances_failed` 로 보고하고 나머지 결과로 degrade.
* 각 결과에는 출처 배지(`source_instance_key/name`)가 붙는다. 로컬 결과는 회색, peer 결과는 인스턴스명.

== 상세 / 샘플 드릴다운

검색·탐색에서 데이터셋 이름을 누르면 federated URN(`{instance_key}::{remote_urn}`)으로 전체화면
상세로 이동한다. 상세 화면은 *로컬 데이터셋 상세와 동일한* 메타데이터 표(라벨·값 한글화 포함)를
재사용한다.

* *상세* — `HYBRID`/`LIVE` 는 원 인스턴스에서 실시간 프록시. 스키마·태그·소유자·DDL·확장 메타.
* *샘플* — 미러에 저장된 샘플이 있으면 오프라인으로, 없으면 LIVE 로 조회
  (xref:federation/harvest.adoc[샘플 미러링] 참고).

== cross-instance 리니지

엔드포인트 URN 은 데이터소스 전역 유일이므로, 로컬 리니지와 여러 peer 가 보고한 미러 리니지를
같은 URN 키로 합쳐(stitch) BFS 그래프를 만든다. 각 노드는 로컬/미러(어느 peer)/미해석으로
해석되고, 각 엣지는 보고 출처(`reported_by`)를 가져 팀 경계를 넘는 데이터 흐름을 한눈에 본다.
