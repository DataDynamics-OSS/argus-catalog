= 인스턴스 관리

페더레이션 > *인스턴스 관리* 탭에서 연합 대상 peer 를 등록·수정·삭제하고 연결 상태를 점검한다.
peer 등록/변경은 관리자(admin) 전용이다.

== 인스턴스 등록

[cols="1,3",options="header"]
|===
| 항목 | 설명
| 식별 키(instance_key) | 허브 내 peer 식별·URN 네임스페이스 접두. *영문 대소문자·숫자·`_`·`-`* 만 허용하며 전역 유일. 등록 후 변경 불가.
| 이름 | 화면 표시용 이름
| Base URL | peer 의 접속 주소. `/health` 와 `/api/v1/federation/export/*` 를 제공하는 카탈로그 API 여야 한다.
| 모드 | `LIVE` / `HARVEST` / `HYBRID` (xref:federation/index.adoc[모드 설명] 참고)
| 동기화 주기(초) | HARVEST 스케줄러가 이 peer 를 가져오는 주기
| 서비스 토큰 | peer 의 export API 가 토큰을 강제할 때 입력(선택)
|===

* *식별 키 중복* — 이미 사용 중인 키면 등록 버튼을 눌렀을 때 입력란 아래에 즉시 에러를
  표시하고, 백엔드도 409 로 거절한다.
* *Base URL 주의* — 프론트엔드(예: `:3000`)가 아니라 카탈로그 *API* 를 가리켜야 한다.
  프론트 프록시가 `/api/v1/*` 와 `/health` 를 백엔드로 전달하면 프론트 주소로도 동작한다.

== 모드 설명

모드 셀렉트는 각 항목에 한 줄 설명을 보여주고, 선택한 모드의 상세 설명을 폼 하단에 표시한다.

* `HYBRID`(권장) — 가져온 데이터로 검색하고, 상세는 원본 인스턴스에서 실시간 조회.
* `HARVEST` — 주기적으로 미러·재임베딩. peer 가 꺼져 있어도 검색 가능(에어갭 친화).
* `LIVE` — 검색 시점에 실시간 fan-out. 항상 최신이나 peer 가 꺼지면 그 결과는 빠짐.

== 상태점검

각 행의 *상태점검* 은 peer 의 `/health` 를 호출해 도달성·지연·버전을 확인한다.

* 성공 — 초록 ✓ `지연(ms) · 버전`
* 실패 — 빨강 `불가 · 사유`(예: `HTTP 404`, `Connection refused`). 결과를 클릭하면 다시 점검한다.

== 노출 항목 선택(소비자)

등록/수정 다이얼로그의 *노출 항목 불러오기* 로 peer 가 advertise 하는 capability 를 가져와,
이 인스턴스 화면에 표시할 항목만 고른다(미선택 시 peer 가 제공하는 전부 표시).

== 삭제

인스턴스를 삭제하면 그 peer 에서 가져온 *미러 데이터셋·임베딩·리니지·가져오기 이력* 이 모두
함께 삭제된다(DB `ON DELETE CASCADE` + 샘플 파일 정리). 원본 peer 의 실제 데이터와 본 카탈로그의
로컬 데이터셋은 영향을 받지 않는다. 삭제 전 확인 다이얼로그로 한 번 더 묻는다.
