= Extension 개요

쿼리 엔진·DBMS 측에 설치해 *메타데이터·쿼리 리니지* 를 자동 수집하는 확장
컴포넌트. 각 컴포넌트는 카탈로그의 `platform_id` 와 연결돼 쿼리 이벤트 또는
스키마 메타데이터를 *Metadata Sync 서비스* (port 4610) 에 비동기 전송한다.

== 공통 설치 흐름

. 카탈로그 UI 에서 *Platforms → Add Platform* 으로 대상 시스템 등록 → `platform_id` 확보.
. Extension 빌드/배포 (각 페이지 참조).
. Extension 설정의 `platformId` 또는 `platform.id` 에 1번에서 얻은 값 입력.
. Metadata Sync 서비스 (`http://<host>:4610`) 가 수신 가능한지 확인.
. 대상 시스템 재시작 후 쿼리 / 동기화 실행으로 수집 검증.

== 인증

* Metadata Sync 와 Extension 사이 통신은 기본적으로 HTTP — 사내망 전제.
* 외부 노출 시 카탈로그의 *Service Account* 토큰을 헤더로 전달하도록 Reverse Proxy 또는
  Sync API 앞단에 인증 게이트웨이 배치 권장.

== 전송 실패 처리

대부분의 Extension 은 *내부 큐* (예: `LinkedBlockingQueue`, daemon thread) 에서 비동기
전송. 큐가 가득 차면 이벤트가 *드롭* 된다. 영구 실패는 로컬 로그에 출력.
