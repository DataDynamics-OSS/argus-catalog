= 데이터셋

데이터셋은 Argus Catalog 의 중심 엔티티다. 플랫폼 전반의 테이블 · 뷰 · 토픽 ·
오브젝트를 단일 카탈로그로 등록하고 검색·조회한다.

== 화면 구성

좌측 사이드바 *Data Catalog → Datasets* 메뉴에서 진입한다.

* *상단 검색바* — 이름·URN·qualified name 부분 일치 검색.
* *필터* — 플랫폼, 환경 (PROD/STAGING/DEV), 태그, 소유자, 도메인.
* *목록* — 이름 / 플랫폼 / 환경 / 태그 / 마지막 동기화 시각 컬럼. 헤더 클릭으로 정렬.
* *행 클릭* — 데이터셋 상세 화면으로 이동.

== 데이터셋 등록

image::catalog-datasets/create.png[데이터셋 등록, 800]

데이터셋은 두 가지 경로로 등록된다.

수동 등록::
*+ Add Dataset* 버튼. 다음 필드를 입력한다.
+
[cols="1,3"]
|===
| 필드 | 설명

| 이름
| `database.table` 또는 `topic` 형식 권장.
| 플랫폼
| 미리 등록된 플랫폼에서 선택. 없으면 xref:catalog/platforms.adoc[플랫폼] 에서 먼저 등록.
| 환경
| PROD / STAGING / DEV.
| 설명
| Markdown 지원. AI 자동 생성으로 비워둘 수도 있음.
| 태그 · 소유자 · 용어
| 등록 후 상세 화면에서 추가 가능.
|===

자동 동기화::
플랫폼 카드의 *Sync* 버튼 한 번으로 해당 플랫폼의 테이블·컬럼이 일괄 등록된다.
MySQL · MariaDB · PostgreSQL 은 직접 연결 기반 자동 수집을 지원한다.
자세한 흐름은 xref:catalog/platforms.adoc#_메타데이터_동기화[플랫폼 → 메타데이터 동기화] 참조.

== URN 네이밍

등록된 데이터셋은 시스템 전역에서 URN 으로 식별된다.

----
{platform_id}.{database}.{table}.{ENV}.dataset
예: mysql-19d0bfe9.sakila.film_text.PROD.dataset
----

* `qualified_name` = `{platform_id}.{database}.{table}`
* `name` = `{database}.{table}` (표시용)

URN 은 리니지 · 알림 · External API 에서 데이터셋을 가리키는 키로 사용된다.

탭 구성은 다음과 같다.

[cols="1,3"]
|===
| 탭 | 내용

| Overview
| 설명 · 환경 · 행 수 · 최신 동기화 시각 · 통계 카드.
| Schema
| ag-grid 기반 인라인 편집. 컬럼 · 타입 · NULL · PK/Unique/Index · 설명.
| Schema History
| 동기화마다 스냅샷 저장. diff 뷰로 추가/삭제/타입변경 표시.
| Tags
| 태그 추가·해제. 부착된 태그는 검색·필터에 반영.
| Owners
| Technical / Business / Data Steward 3종 역할.
| Sample Data
| 소스 DB 에서 수집한 샘플 행 미리보기 (Parquet 캐시).
| Quality
| 프로파일 결과, 규칙 평가, 품질 점수 추세. 상세는 xref:catalog/quality.adoc[].
| Terms
| 표준 용어 매핑. 데이터 표준 사전과 연결. 상세는 xref:catalog/standards.adoc[].
| Lineage
| 업/다운스트림 데이터셋 그래프. 컬럼 매핑. 상세는 xref:catalog/lineage.adoc[].
| Pipelines
| 입력/출력으로 연결된 파이프라인 (Airflow DAG, Kestra Flow, NiFi Flow 탭).
| Comments
| 중첩 답글 토론 스레드.
|===

== 검색과 필터

목록 화면의 필터 외에 전역 헤더의 *전역 검색* 또는 xref:ops/search.adoc[검색] 페이지에서
키워드 + 시맨틱 하이브리드 검색이 가능하다. 임베딩이 켜져 있으면 자연어 질의로
"고객 거래 내역" 같은 표현으로도 데이터셋이 검색된다.

== 수정·삭제

* *수정* — Overview 헤더의 편집 아이콘. 이름·설명·환경·플랫폼 변경.
* *삭제* — Overview 우측 메뉴. 소프트 삭제 후 관련 리니지·태그·댓글이 함께 비활성화된다.
* *권한* — `argus-admin` 또는 데이터셋 소유자로 등록된 사용자만 수정·삭제 가능.

== AI 자동 메타데이터

LLM 이 활성화된 환경에서는 다음 메타데이터를 자동 생성할 수 있다.

* 테이블 설명 / 컬럼 설명 일괄 생성
* 태그 추천 (기존 태그 활용 + 새 태그 제안)
* PII 탐지 (EMAIL · PHONE · SSN · NAME · ADDRESS)

미리보기 (apply=false) 로 결과 확인 후 적용 (apply=true). 동기화 시 자동 생성
옵션을 켜면 빈 설명이 자동으로 채워진다.
