= 태그·용어집

데이터셋 분류와 비즈니스 용어 정의를 위한 두 가지 도구.

태그가 *분류·발견* 을 위한 단순 라벨이라면, 용어집은 *조직의 공식 용어 체계*
를 정의하는 계층 구조다.

=== 데이터셋에 태그 부여

데이터셋 상세 *Tags* 탭에서 검색·선택. 다중 태그 부여 가능. 부착된 태그는
다음 위치에 즉시 반영된다.

* 데이터셋 목록·검색 필터
* xref:ops/alerts.adoc[알림] 규칙의 Scope=TAG 평가

=== CATEGORY / TERM 계층

용어집은 두 단계 트리다.

* *CATEGORY* — 도메인 영역 (예: 고객, 결제, 마케팅). 하위 노드를 가질 수 있음.
* *TERM* — 실제 용어 (예: 고객 등급, 결제 수단). 데이터셋·컬럼과 매핑되는 대상.

깊이는 임의로 중첩 가능하나 *조회 편의를 위해 2~4 단계를 권장*.

=== 용어 등록

각 노드는 다음을 가진다.

* *이름* — 용어 (예: "고객 등급").
* *정의* — 비즈니스 정의. Markdown 지원.
* *동의어* — 다중 입력. 검색 매칭 폭 확대.
* *연관 데이터셋·컬럼* — 자동 또는 수동 매핑.

=== Move to 기능

트리 노드를 다른 부모로 이동한다. 우클릭 메뉴 → *Move to → \<target>* 또는
드래그-드롭. 하위 트리가 함께 이동하며 이동 후 URN/식별자는 유지된다.

=== 데이터셋·컬럼 매핑

데이터셋 상세의 *Terms* 탭에서 용어를 검색해 매핑. 컬럼 단위 매핑은
스키마 그리드의 *Term* 컬럼에서 부여한다.

자동 매핑은 xref:catalog/standards.adoc#_자동_매핑[데이터 표준 → 자동 매핑] 에서 일괄 실행
할 수 있다.
